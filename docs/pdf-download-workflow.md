# PDF 下载页工作流

这份说明记录仓库里 PDF 下载与浏览页的维护方式，目标是把工作流压缩成“把 PDF 放到固定目录，然后重新启动开发服务器或重新构建即可”。

## 目录约定

- PDF 投放目录：public/pdfs/
- 页面路由：/pdfs/
- 导航入口：Exhibition -> PDF Library

不要再把需要公开访问的 PDF 直接放到仓库根目录。根目录文件不会被 Astro 当成静态资源公开，页面也不会自动发现它们。

## 日常使用

1. 把一个或多个 PDF 复制到 public/pdfs/。
2. 保持文件名尽量清晰，页面会直接根据文件名自动生成标题。
3. 重新运行 pnpm dev，或在开发服务器运行中重启一次。
4. 打开 /pdfs/ 检查列表、页内预览、下载按钮和新窗口打开是否正常。

## 页面行为

- 页面会在构建阶段直接扫描 public/pdfs/。
- 只识别 .pdf 文件，其他文件会被忽略。
- 列表默认按文件最后修改时间倒序排列。
- 每个条目都提供：页内预览、下载、新窗口打开。
- 中文文件名和包含空格的文件名可用，链接会自动编码。

## 维护建议

- 如果文件名只用于内部区分，发布前建议改成更适合展示的名字。
- 如果后续需要摘要、封面或作者字段，优先扩展为“同名可选元数据文件”，不要先改成强制手写配置。
- 如果将来 PDF 明显增多，再考虑分类、分页或搜索；当前实现刻意保持轻量。
